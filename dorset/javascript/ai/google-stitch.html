Introduction to Stitch 2.0

Google Stitch 2.0 is a sophisticated, browser-based design environment powered by Gemini models. It is specifically engineered to eliminate the visual inconsistency often referred to as “AI slop”. By providing an aesthetic guardrail for front-end development, Stitch 2.0 allows designers and developers to generate high-fidelity UI/UX mockups that surpass the default output of standard coding agents.

The Design-to-Code Workflow

Stitch 2.0 does not replace agentic coding tools like Claude Code. Rather, it serves as the essential visual foundation in a professional pipeline. The recommended workflow follows a three-stage progression:

  • Inspiration: Sourcing visual references from the web.
  • Stitch 2.0: Synthesising these references into an “80–90% solution” for the front-end design.

By resolving the visual layer within Stitch first, developers significantly reduce token expenditure in subsequent steps. Coding agents no longer need to perform expensive, iterative CSS “guesswork” to achieve a professional look.

Addressing the "AI Slop" Problem

A recurring issue with fully automated coding agents is the production of uninspired, "samey" user interfaces. Stitch 2.0 mitigates this by:

  • Decoupling Design from Logic: Allowing developers to iterate on aesthetics in a dedicated visual environment before committing to code.
  • Leveraging Visual Inspiration: Using screenshots and URLs to guide the AI toward specific, high-quality aesthetic styles.
  • Providing a Comprehensive Design System: Automatically generating a "creative Northstar" that defines colors, typography, and component rules to ensure consistency.

The Optimized Web Design Workflow

Here is a high-efficiency workflow for modern web development that combines the strengths of visual AI and agentic coding agents.

  1. Inspiration Phase: Identify a high-quality design on Pinterest or similar platforms and take a screenshot.
  2. Design Phase (Stitch 2.0):
    • Upload the screenshot to Stitch.
    • Provide a prompt specifying the desired style and layout (e.g., "Create a landing page in the style of the screenshot").
    • Stitch uses a background process (noted as "Nano Banana") to mimic the screenshot’s visual language.
    • Refinement Phase: Use the "Variants" and "Explore" tools to iterate on the layout. Utilize "Live Mode" for specific adjustments like background motion.
  3. Export Phase: Access the "More" menu and use "Export to Clipboard" to grab the generated front-end code.
  4. Implementation Phase (Claude Code): Open Claude Code. Provide the exported code as a reference. Instruction: "Create a landing page for [Project Name], here is the front-end code: [Paste Code]."
  5. Polishing Phase: Once the foundation is set in Claude Code, use tools like 21st.dev to refine UI components like buttons and advanced animations.

Conversational Initialisation and Project Setup

Project creation begins via a Natural Language UI (NLUI) entry point. This conversational initialisation allows users to define the project scope through simple descriptive prompts.

Project Configuration

Users must select their project target and the underlying model engine:

  • Project Type Selection: Choose between App (mobile-first, native-style interfaces) or Web App (responsive desktop and browser-focused layouts).
  • Model Engine Selection: Stitch leverages Google’s frontier models to drive the design logic.

The Inspiration Input Engine

Stitch 2.0 uses Nano Banana (and Nano Banana Pro) as its backend image processing engine to interpret visual references. Users can provide context through:

  • File Uploads: Uploading screenshots for the model to analyse and mimic.
  • Direct URLs: Providing live website addresses for the model to crawl.

Recommended Sources:

For optimal landing page design, the following platforms are recommended:

  • Dribbble — for high-concept UI trends.
  • Godly — for curated, high-end web aesthetics.
  • Pinterest — particularly effective for landing page layouts. Once a preferred style is identified, Pinterest’s recommendation engine is excellent for sourcing additional, similar references.

The Infinite Canvas Layout

The core workspace is an Infinite Canvas: a spatial environment where multiple design iterations can be viewed and compared simultaneously, without the constraints of a traditional paginated interface.

  • Agent Log: Located in the bottom-left corner, the Agent Log provides a transparent view of the model’s “thinking” process and command history. It displays how the AI interprets prompts and translates them into design instructions.
  • Edit Mode: Selecting the Edit (pencil icon) button enables direct manipulation of individual UI components or global layout adjustments.
  • Preview Mode: This mode renders the design in a new browser tab, allowing for a full-scale interactive assessment of the prototype’s responsiveness and flow.

The Design System Panel: Obsidian Ember

Stitch 2.0 generates a centralised design system for every project — often titled Obsidian Ember by default — which acts as the visual bedrock for all generated components.

The Design System panel allows for granular control over the following configurable visual parameters:

  • Seed Colors: Definition of Primary, Secondary, Tertiary, and Neutral palettes.
  • Typography: Centralised font selection and hierarchy settings.
  • Corner Radius: Global control over UI “roundness,” from sharp professional edges to modern, pill-shaped components.

The design.md File: The Creative Northstar

The design.md file is a synthesised markdown document that serves as the project’s strategic blueprint. It is the primary mechanism for avoiding “AI slop.” Even when provided with a vague prompt, Stitch 2.0 uses this file to establish:

  • Overview & Creative Northstar: The high-level aesthetic intent.
  • Colour & Typography Strategy: Logic-based rules for why specific visual choices were made.
  • Template Divergence: Explicit instructions for the model to break away from generic, “standard” AI templates and create a unique, brand-aligned identity.

Variant Generation and Iteration Tools

Stitch provides a robust suite for creative exploration, allowing users to rapidly pivot designs without restarting the project.

The iteration workflow operates as follows:

  • Right-Click Interaction: Right-clicking any layout or component opens a context menu with Regenerate (a total redo of the element) and Variants.
  • Granular Variants: Users can choose to vary specific dimensions — such as Layout, Colour Scheme, or Images — while keeping the rest of the design intact.
  • Custom Exploration: The “Explore” feature includes a Creative Range slider (reminiscent of Midjourney’s ‘stylize’ parameters). Users can request multiple options (e.g. three versions) and determine how far the model should deviate from the original reference.

Reference Docking

When the model generates multiple variations, clicking a version adds it to a reference dock at the bottom of the canvas. This signals to the model that future refinements should prioritise the characteristics of those specific selected references.

Live Mode: Screen-Aware Interaction

Live Mode transforms the design process into a real-time, voice-activated collaboration. When activated, the model “looks” at the user’s active screen, providing immediate context for voice commands.

Users can request complex, dynamic modifications through natural conversation. For example, a user can say: “Add some dynamic motion graphics and subtle cursor effects.” Stitch 2.0 can implement sophisticated features such as dynamic background dots that move away from the user’s mouse cursor in real-time.

Code Export and Developer Handoff

Once a design reaches the “80–90% solution” stage — where the visual identity is locked and the layout is professional — it is ready for handoff to a development environment.

The handoff workflow is as follows:
