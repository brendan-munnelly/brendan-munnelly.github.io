Form layout and HTML5 inputs

Forms are the primary way web applications collect data from users. Before writing JavaScript, it is important to understand how to structure your HTML.

Layout Conventions

There are two main conventions for placing text labels next to input fields:

  • Labels to the Left: Often used on wide desktop screens.
  • Labels Above: The modern standard for mobile responsiveness, as it prevents horizontal scrolling on small screens. You can easily achieve this by setting your <label> to display: block; in CSS.

HTML5 Input Types: text vs email

Notice the difference between the two inputs below. One is type="text" and the other is type="email".

<form id="registerForm" class="label-above">
    <!-- Standard Text Input -->
    <div class="form-group">
        <label for="userName">Full Name:</label>
        <input type="text" id="userName" name="userName" required>
        <span id="nameError" class="error-text">Name is required.</span>
    </div>

    <!-- Email Input -->
    <div class="form-group">
        <label for="userEmail">Email Address:</label>
        <input type="email" id="userEmail" name="userEmail" required>
        <span id="emailError" class="error-text">Please enter a valid email.</span>
    </div>

    <button type="submit">Register</button>
</form>

Using type="email" provides two massive "free" benefits before you even write any JavaScript:

  1. Mobile UX: On smartphones, it automatically opens an email-optimized keyboard showing the @ and .com keys.
  2. Built-in Validation: The browser will automatically pause the form submission and warn the user if they forget the @ symbol!

The submit event & default behavior

By default, when a user clicks a Submit button, the browser attempts to send the data to a server and refreshes the page. In modern JavaScript applications, we usually want to stop this page refresh so we can validate and handle the data ourselves.

We do this using the event object's .preventDefault() method.

const myForm = document.getElementById("registerForm");

myForm.addEventListener("submit", e => {
    // 1. Stop the page from refreshing!
    e.preventDefault(); 
    
    console.log("Form submitted successfully!");
});

The "Enter" Key Freebie: Because you attached the listener to the submit event of the form (rather than just the click event of the button), HTML automatically handles keyboard accessibility. If the user presses the "Enter" key while typing in any text box, it will trigger this exact same event!

Extracting values and validation

To get the text a user typed into an input, we read its .value property. We also use the .trim() string method to remove any accidental blank spaces at the start or end of their text.

myForm.addEventListener("submit", e => {
    e.preventDefault(); 
    
    const nameInput = document.getElementById("userName");
    
    // Extract the value and remove accidental spaces
    const cleanName = nameInput.value.trim();

    // Basic Validation: Is it empty?
    if (cleanName === "") {
        document.getElementById("nameError").style.display = "block";
    } else {
        document.getElementById("nameError").style.display = "none";
        console.log(`Welcome, ${cleanName}!`);
        myForm.reset(); // Clears all fields
    }
});

Real-time feedback: blur & input

Waiting until the user clicks "Submit" to show them an error can be frustrating. Modern websites provide immediate feedback as you navigate the form using two key events:

  • The blur event: Fires the exact moment the user clicks or tabs away from an input field.
  • The input event: Fires instantly on every single keystroke.
const emailInput = document.getElementById("userEmail");

// BLUR: Check if they left the email field blank after clicking away
emailInput.addEventListener("blur", () => {
    if (emailInput.value.trim() === "") {
        emailInput.style.border = "2px solid red";
    } else {
        emailInput.style.border = "2px solid green";
    }
});

// INPUT: Provide instant feedback as they type (e.g., character count)
const nameInput = document.getElementById("userName");

nameInput.addEventListener("input", () => {
    console.log(`You have typed ${nameInput.value.length} characters.`);
});

Keyboard events: keydown

While the "Enter" key is handled automatically by the form submit, you can use the keydown event to listen for other specific keys to improve the user experience. For example, allowing the user to clear the form by pressing the "Escape" key.

// Listen for keystrokes anywhere on the page
document.addEventListener("keydown", e => {
    // Check which key was pressed
    if (e.key === "Escape") {
        myForm.reset();
        console.log("Form cleared via Escape key.");
    }
});
