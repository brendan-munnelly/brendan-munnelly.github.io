Introduction

Below you can see a screenshot of the single-page website you will create.

  • In the <header> or 'hero block' is a full-width background image.
  • In the <main> part are five <section> blocks of content.   The first is a 'deckhead' with a single-column layout.   The remaining four are blocks for each of the four smoothie products. All have a two-column split layout.
  • All text on the web page is in the Neucha font from Google Fonts.

Project folders and files

Your first task is to create the folders you need for this project.

  1. Inside the 📁 portfolio of your ‘main’ 📁 websites folder, create a sub-folder to hold the files for your Smoothies project.   Name the new sub-folder as 📁 smoothies (all small letters).
  2. Inside this new folder, create an 📁 assets sub-folder.   And inside this, create a 📁 CSS sub-folder and an 📁 img sub-folder.

✅  All done. You now have created the folder with sub-folders for your Smoothies web page project.

Creating the deckhead layout

Now it's time to create a deckhead/standfirst section that will be positioned under the hero block.

Working with the deckhead in Lunevedy

Follow these steps:

  1. On the Home page of the Lunevedy app, click the Single column option.
  2. On the Section menu, leave the Section Class Name option at section-selector-1 and centre-align the content for both desktop and mobile screens.
  3. On the Text menu, select the options below.
  4. On the Illustrations menu, remove the illustration.

When finished, click the HTML button near the bottom-right of the screen.

Working with the deckhead in VS Code

Follow these steps:

  1. Switch to VS Code and display the index.html web page. Inside the opening <main> tag pair, select the first <section> block. Now paste in your copied content.
  2. Delete the h2 sub-heading, and the second and third p paragraphs of sample text. Your VS Code screen should now look as shown below.
  3. Update the deckhead with the words from the text.html file.
  4. Return to the Lunevedy app and click the CSS button at the bottom right.
  5. In VS Code, display the custom.css stylesheet, and paste in the copied CSS style rule. See below.
  6. Save your index.html and custom.css files and display the web page in your browser. It should look as shown below.

Uploading your project folder to GitHub

After finishing your project, you are now ready to upload it to your account on GitHub.

  1. Open a new tab in your web browser and go to GitHub.com. If you are not already signed in to your GitHub account, sign in now.
  2. On your GitHub home page, click the ‘repo’ that holds your web pages. Its name will look as follows, where username is your chosen username on GitHub.   username.github.io
  3. On the next GitHub screen displayed, click the portfolio folder.
  4. With your portfolio folder displayed on GitHub, click Add file button.
  5. From the dropdown list displayed, choose the option Upload files.
  6. In File Explorer (Windows 10) or Finder (Apple Mac), drag-and-drop your üìÅ smoothies folder to upload it to the portfolio folder within your repository on GitHub.
  7. Scroll down to the bottom of the GitHub screen, and accept or edit the short message (Add files via upload) in the Commit changes box.
  8. Finally, click the green Commit changes button to upload your files.

Your portfolio project web page is now published on GitHub at a web address similar to the following:

https://username.github.io/portfolio/smoothies/index.html

or simply:

https://username.github.io/portfolio/smoothies

It may take a few minutes for your uploaded files to appear on GitHub.
